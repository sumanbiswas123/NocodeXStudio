# OneView SDK (`window.platform`) - Developer Guide

This guide is only for the app SDK exposed as `window.platform` inside OneView webview apps.

## Quick Start

```js
async function init() {
  const caps = await window.platform.capabilities.get();
  if (!caps.fs?.read || !caps.fs?.write) {
    throw new Error("File system capability not available");
  }

  const env = await window.platform.env.get();
  console.log("Running on:", env.platform, env.hostname);
}
```

## SDK Surface

`window.platform` provides:

- `platform.fs.readDir(path)`
- `platform.fs.readFile(path, options?)`
- `platform.fs.writeFile(path, content)`
- `platform.fs.createDir(path)`
- `platform.fs.exists(path)`
- `platform.dialog.showOpenDialog(options)`
- `platform.dialog.showSaveDialog(options)`
- `platform.dialog.showMessageBox(options)`
- `platform.env.get()`
- `platform.env.getPath(name)`
- `platform.capabilities.get()`

## File System Operations

### 1. Check if path exists

```js
const exists = await window.platform.fs.exists("C:\\Users\\SumanBiswas\\Documents\\notes.txt");
console.log("Exists:", exists);
```

### 2. Create directory (recursive)

```js
await window.platform.fs.createDir("C:\\Users\\SumanBiswas\\Documents\\OneViewData\\reports");
```

### 3. Write file

```js
const filePath = "C:\\Users\\SumanBiswas\\Documents\\OneViewData\\reports\\today.txt";
await window.platform.fs.writeFile(filePath, "Report generated by OneView SDK");
```

### 4. Read text file (UTF-8 default)

```js
const text = await window.platform.fs.readFile(
  "C:\\Users\\SumanBiswas\\Documents\\OneViewData\\reports\\today.txt"
);
console.log(text);
```

### 5. Read file with explicit encoding

```js
const text = await window.platform.fs.readFile(
  "C:\\Users\\SumanBiswas\\Documents\\OneViewData\\reports\\today.txt",
  { encoding: "utf8" }
);
```

### 6. List directory content

```js
const items = await window.platform.fs.readDir("C:\\Users\\SumanBiswas\\Documents\\OneViewData");
/*
items = [
  { name: "reports", isDirectory: true, isFile: false },
  { name: "readme.txt", isDirectory: false, isFile: true }
]
*/
console.log(items);
```

### 7. Common pattern: create-if-missing then write

```js
async function writeJsonSafe(dir, fileName, data) {
  const dirExists = await window.platform.fs.exists(dir);
  if (!dirExists) await window.platform.fs.createDir(dir);

  const path = `${dir}\\${fileName}`;
  await window.platform.fs.writeFile(path, JSON.stringify(data, null, 2));
  return path;
}
```

### 8. Common pattern: read JSON file

```js
async function readJson(path) {
  const raw = await window.platform.fs.readFile(path, { encoding: "utf8" });
  return JSON.parse(raw);
}
```

## Accessing Useful System Paths

Use `platform.env.getPath(name)` instead of hardcoding paths when possible.

```js
const documents = await window.platform.env.getPath("documents");
const appData = await window.platform.env.getPath("appData");
const temp = await window.platform.env.getPath("temp");

console.log({ documents, appData, temp });
```

Typical keys:

- `home`
- `appData`
- `userData`
- `temp`
- `downloads`
- `documents`

## Dialog Operations

### Open file/folder picker

```js
const result = await window.platform.dialog.showOpenDialog({
  title: "Select input file",
  properties: ["openFile"],
  filters: [{ name: "Text Files", extensions: ["txt", "md", "json"] }]
});

if (!result.canceled) {
  console.log("Selected:", result.filePaths[0]);
}
```

### Save dialog + write file

```js
const save = await window.platform.dialog.showSaveDialog({
  title: "Save report",
  defaultPath: "report.txt",
  filters: [{ name: "Text", extensions: ["txt"] }]
});

if (!save.canceled && save.filePath) {
  await window.platform.fs.writeFile(save.filePath, "Saved from OneView app");
}
```

### Message box

```js
await window.platform.dialog.showMessageBox({
  type: "info",
  title: "Operation complete",
  message: "File created successfully."
});
```

## End-to-End Example (Directory + File CRUD Flow)

```js
async function runExample() {
  const documents = await window.platform.env.getPath("documents");
  const baseDir = `${documents}\\OneViewSDKDemo`;
  const dataFile = `${baseDir}\\data.json`;

  // 1) ensure directory
  if (!(await window.platform.fs.exists(baseDir))) {
    await window.platform.fs.createDir(baseDir);
  }

  // 2) write JSON
  const payload = {
    createdAt: new Date().toISOString(),
    items: ["alpha", "beta", "gamma"]
  };
  await window.platform.fs.writeFile(dataFile, JSON.stringify(payload, null, 2));

  // 3) read back
  const raw = await window.platform.fs.readFile(dataFile, { encoding: "utf8" });
  const parsed = JSON.parse(raw);
  console.log("Parsed:", parsed);

  // 4) list directory
  const entries = await window.platform.fs.readDir(baseDir);
  console.log("Entries:", entries);
}
```

## Error Handling Best Practice

```js
async function safeRead(path) {
  try {
    if (!(await window.platform.fs.exists(path))) return null;
    return await window.platform.fs.readFile(path, { encoding: "utf8" });
  } catch (err) {
    console.error("Read failed:", err.message || err);
    return null;
  }
}
```

## Notes

1. Paths in Windows should use escaped backslashes in JS strings (`\\`).
2. `readFile` defaults to UTF-8; pass `options.encoding` explicitly if needed.
3. `readDir` returns names and type flags, not full absolute paths.
4. For app portability, prefer `env.getPath(...)` over hardcoded absolute paths.
